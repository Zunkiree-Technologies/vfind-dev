# V-Find-Stark Memorize Command

Save the current session's progress to the brain system.

## What to Update

### Always Update
1. **`.brain/status-current.md`**
   - Update "Last Updated" timestamp
   - Update current focus
   - Move completed items
   - Add new blockers if any

2. **`.brain/progress-journal.md`**
   - Add new entry with date
   - What was accomplished
   - Decisions made
   - Problems encountered and solutions

### Update If Changed
3. **`.brain/next-session.md`**
   - Update priorities based on current session
   - Add new pending items discovered

4. **`.brain/decisions-log.md`**
   - Log any significant architectural decisions
   - Document rationale and impact

5. **`.brain/project-context.md`**
   - Update if tech stack changed
   - Add new features discovered/implemented

## Sync to Global (If Significant)

If this session produced patterns useful across projects:
- Note them for manual sync to Global Sadin-Stark
- Cross-project patterns go in parent brain

## Format for Journal Entry

```markdown
## YYYY-MM-DD - Brief Title

**Session:** Description of session focus
**Duration:** Approximate time

### Accomplished
- Item 1
- Item 2

### Decisions Made
- Decision and rationale

### Problems Solved
- Problem: Solution

### Next Steps
- Priority items for next session
```

## After Memorizing

Confirm what was saved and provide a handoff summary for the next session.
